spark.spec-tacular.restify

csv-build

(csv-build sp-name sp-list)

returns a comma delimited string listing all instances of a given spec. usage like (build-csv :User (sd/get-all-of-type (db/get-db) (sp/get-spec :User)))

csv-extract-fields

(csv-extract-fields spec-name sp)

returns list of pairs of colName and colValues, where colName is a list of ‘item’ names representing the nesting structure. eg ([(:phones) ‘List of 0 phones’] [(:first-name) nil] [(:some-nested-thing :first-name) nil])

explicitly-tag

(explicitly-tag sp)

deep-walks a sp object adding explicit :spec-tacular/spec spec name tags to the object and all its child items.

from-json-friendly

(from-json-friendly spec-name jf)

converts from the json converted rep to the given spark record re-namespacing the spec tag :spec-tacular-spec to :spec-tacular/spec. Also converts the spec tag back into a keyword, (json would have stringed it)

make-expanded-routes

(make-expanded-routes route-above spec get-conn-ctx-fn)

Creates a list of routes for a RESTful API for the given spec. Allows for get/post on the collection and get/put/delete on individual resources.

make-routes

(make-routes route-above parent-route spec get-conn-ctx-fn & [simple-repr-fn])

Creates a list of routes for a RESTful API for the given spec. Allows for get/post on the collection and get/put/delete on individual resources. expects body-params, html-body and json-body interceptors.

to-json-friendly

(to-json-friendly sp)

converts sp object to json representation with explicit spec tags with the un-namespaced :spec-tacular-spec keyword
